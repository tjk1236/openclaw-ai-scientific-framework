## 🎯 核心能力

### 1. 分镜生成系统 (3x3框架 + 强制审查)

**工作流程:**
```
小说内容输入
    ↓
角色档案生成 (角色 + 声线)
    ↓
图片提示词生成 (9张/组)
    ↓
视频提示词生成 (含台词时间轴)
    ↓
【🔴 强制审查员介入】逻辑校对+评分
    ├─ 评分≥90 → 直接输出
    ├─ 评分70-89 → 自动修正+输出
    └─ 评分<70 → 标记问题+提供修正版+记录到错误案例库
    ↓
九宫格输出 (含衔接镜 + 审查报告)
```

**强制审查规则:**
- 所有生成分镜 **必须** 经过审查员检查
- 审查报告作为分镜的必要组成部分输出
- 评分<70的案例自动进入 `video-learning/errors/` 案例库
- 每周基于案例库更新审查规则

**输出格式:**
- 图片提示词 (英文 + 中文对照)
- 视频提示词 (自然语言 + 镜头运动)
- 台词声线表 (时间戳 + 角色 + 声线描述)
- **审查报告** (评分+问题+修正建议)
- 九宫格布局图

### 2. 提示词库

**景别标签:**
| 中文 | 英文 | 用途 |
|------|------|------|
| 极远景 | Extreme Wide Shot | 场景建立 |
| 远景 | Wide Shot | 环境交代 |
| 全景 | Full Shot | 全身展示 |
| 中景 | Medium Shot | 对话标准 |
| 近景 | Medium Close-up | 表情细节 |
| 特写 | Close-up | 情绪强调 |
| 极特写 | Extreme Close-up | 眼神/细节 |

**必须包含标签:**
```
"无时间码，无字幕"
"8K超高清，极致细节，电影级质感"
```

**⚠️ 重要规则: 提示词不得缩略**
- **不节省Token**: 视频生成提示词必须完整详尽，不得为省Token而删减
- **缩略提示词会严重影响视频质量**: AI视频工具依赖详细描述理解场景
- **标准**: 单条视频提示词至少50-100中文字，复杂场景200+字
- **必须包含**: 主体描述 + 环境氛围 + 光影风格 + 镜头运动 + 质量标签
- **语言**: 使用中文提示词，AI视频工具对中文支持良好

**镜头运动描述:**
- 缓慢推近、缓慢拉远、缓慢平移
- 快速推近、跟随移动、稳定静止

### 5. 审查员系统 (逻辑校对)

**核心职责:**
校对分镜提示词的逻辑一致性，发现并修正逻辑错误。

**审查维度:**

| 维度 | 检查内容 | 常见错误 |
|------|----------|----------|
| **动作连贯** | 人物动作是否自然过渡 | 上一秒站着，下一秒坐下(无过渡) |
| **场景连贯** | 地点切换是否有交代 | 上一秒在城中，下一秒在家里 |
| **时间连贯** | 时间线是否合理 | 白天→黑夜(无过渡) |
| **服装连贯** | 服装是否保持一致 | 上一秒穿西装，下一秒穿T恤 |
| **人物位置** | 空间位置是否合理 | 两人距离忽远忽近 |
| **道具连贯** | 物品出现/消失是否合理 | 手里突然多出物品 |
| **情绪连贯** | 情绪转变是否自然 | 愤怒→开心(无过渡) |

**评分标准:**

| 等级 | 分数 | 说明 | 处理方式 |
|------|------|------|----------|
| 🟢 优秀 | 90-100 | 逻辑完全通顺，可直接使用 | 直接输出 |
| 🟡 良好 | 70-89 |  minor issues，建议微调 | 输出+修改建议 |
| 🟠 及格 | 50-69 | 有明显逻辑问题，需修改 | 输出问题清单+修正版 |
| 🔴 不及格 | <50 | 逻辑混乱，需重新生成 | 拒绝输出，要求重新输入 |

**审查流程:**
```
生成分镜提示词
    ↓
【审查员介入】逐项检查
    ↓
发现问题 → 标记+说明+建议修正
    ↓
输出评分报告 + 修正后版本
```

**输出格式:**
```
━━━━━━━━━━━━━━━━━━━━
📋 审查报告 - 分镜组A
━━━━━━━━━━━━━━━━━━━━

🎯 综合评分: 78/100 (🟡 良好)

❌ 发现问题:

[问题1] 动作不连贯
  位置: A-3 → A-4
  问题: A-3中男主"转身面对镜头"，A-4中女主"从身后出现"
  修正: A-4改为"女主从门口走入，男主转身看向她"

[问题2] 场景跳跃
  位置: A-6 → A-7
  问题: A-6在"办公室"，A-7突然到"咖啡厅"
  修正: A-7改为"办公室茶水间"或添加转场

[问题3] 服装不一致
  位置: A-2 → A-5
  问题: A-2女主穿"红裙"，A-5变成"白裙"
  修正: 统一为"红裙"

━━━━━━━━━━━━━━━━━━━━
✅ 修正后分镜:
[输出修正后的完整分镜]
━━━━━━━━━━━━━━━━━━━━
```

**审查指令:**
- `审查 [分镜内容]` - 对分镜进行逻辑审查
- `审查 文件 [路径]` - 审查文件中的分镜
- `修正 [问题编号]` - 按建议自动修正

## 🧠 学习进化系统

### 学习目标
持续优化分镜质量和审查能力，从错误中学习，从优秀案例中提取模式。

### 学习资产库

| 目录 | 用途 | 更新频率 |
|------|------|----------|
| `video-learning/errors/` | 错误案例库(评分<70) | 实时 |
| `video-learning/patterns/` | 优秀模式库(评分≥90) | 每周 |
| `video-learning/rules/` | 审查规则更新日志 | 每周 |
| `video-learning/insights/` | 洞察分析报告 | 每月 |

### 学习机制

#### 1. 错误案例学习 (实时)

**收集触发:**
- 审查评分 < 70 的案例自动入库
- 用户手动标记"有明显问题"的案例

**案例结构:**
```json
{
  "id": "ERR-20260225-001",
  "timestamp": "2026-02-25T20:00:00",
  "score": 65,
  "problem_type": "场景跳跃",
  "original": "A-3在办公室... A-4在咖啡厅...",
  "corrected": "A-3在办公室... A-4走出办公室...",
  "root_cause": "缺少场景转换交代",
  "lesson": "场景切换需要过渡镜头或明确交代"
}
```

#### 2. 优秀模式提取 (每周)

**提取来源:**
- 审查评分 ≥ 90 的案例
- 用户标记"非常满意"的案例
- 实际制作后效果好的分镜

**模式类型:**
- 镜头衔接技巧
- 情绪转换手法
- 空间维持方法
- 时间过渡技巧

#### 3. 规则迭代 (每周)

**更新流程:**
```
分析本周错误案例
    ↓
识别新的错误模式
    ↓
补充到审查规则
    ↓
更新评分权重
    ↓
测试验证
    ↓
发布新版本规则
```

**规则版本:**
- v1.0 (2026-02-25): 初始7维度审查
- v1.1 (每周更新): 基于案例库迭代

#### 4. 技能学习联动

**学习管家协作:**
```
视频管家识别知识缺口
    ↓
向学习管家提交需求
    ↓
学习管家优先学习相关技能
    ↓
技能应用到视频管家系统
```

**优先级技能:**
- `prompt-engineering` - 提示词工程优化
- `storytelling` - 叙事结构学习
